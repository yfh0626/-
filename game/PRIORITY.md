# Priority Order (π)

```
π = (state, operation)
```

## Structure

```
0.x    Pre-control — 最优先结算，在控制效果之前
1.x    Control — 沉默、诅咒、封禁
2      Energy — 能量充盈、技能统一能量消耗
─── 3.0 通用位移 ───
3.5    通用位移统一裁决
─── 3.6+ 特殊位移 ───
─── 4.0 通用防御 ───
─── 4.5+ 特殊防御 ───
─── 5.0 通用攻击 ───
─── 5.1+ 特殊攻击 ───
5.5    回合末触发
5.9    反击姿态
5.95   持续状态回合末递减/清理（例如幻鳞蜥客隐身显形）
5.99   不死判定
6      死亡 + draw + countdown + 清理
```

**归类规则**：每一类内通用技能先于特殊技能。
**位移规则**：通用位移完成 → 统一裁决（收集→冲突检测→移动）→ 特殊位移逐个执行。

## Rules

1. 同 priority 同 actor：每回合只提交一个行动，不会自冲突。
2. 同 priority 异 actor：按 actor_A → actor_B 顺序执行。副作用互不交叉（各读各的 target）。伤害累加，顺序无关。
3. 位移冲突：统一裁决中目标冲突 → 都取消；目标被未移动角色占据 → 取消。
4. 沉默互锁：同时沉默双方均生效。
5. 防御值机制已删除：所有角色等效防御值为 0，普通伤害直接造成全额伤害。
6. 蚀骨催毒 → 毒伤：5.21 蚀骨先执行设置覆盖标记，5.21 毒伤读到 → 跳过本次。
7. 血誓再次(5.23)：自身 HP 取此刻已结算的值。
8. special_state：特殊技能状态存 character.special_state。沉默不影响 special_state。
